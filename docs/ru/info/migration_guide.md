## ⚠️ Критическое уведомление о миграции

> **Обязательно прочитайте перед началом миграции.**

Миграция XC_VM переносит **только данные**.
**Вся конфигурация намеренно исключена из процесса миграции.**

Это включает (но не ограничивается):

* API-ключи (например, **TMDb**)
* Учётные данные внешних сервисов
* Настройки, зависящие от окружения
* Конфигурацию панели и системы
* Состояние выполнения и состояние стримов

Все эти значения **необходимо настроить вручную после миграции**.

Это **осознанное архитектурное решение**, а не ограничение и не ошибка.
Пропуск повторной настройки **приведёт к неработающему получению метаданных, обновлению названий стримов и другим связанным функциям**.

---

## ⚙️ Перед началом

> 💡 **Рекомендация:**
> Выполняйте миграцию на **чистой установке XC_VM**.

> ⚠️ **Важно:**
> Системные настройки и настройки панели **НЕ переносятся**.
> Переносятся только данные базы, поддерживаемые механизмом миграции.

Если вы выполняете миграцию в **существующую установку**, учтите следующее:

* XC_VM **удалит все таблицы** в основной базе данных, совпадающие с данными из базы миграции.
* **Резервные копии обязательны.** Автоматического отката не предусмотрено.

## 🚀 Шаги миграции

### 1. Загрузка бэкапа

Загрузите резервную копию существующей базы данных на сервер XC_VM с помощью **SFTP**.

Пример пути:

```text
/tmp/backup.sql
```

---

### 2. Восстановление бэкапа в базу миграции

Очистите базу миграции и восстановите бэкап:

```bash
sudo /home/xc_vm/bin/php/bin/php /home/xc_vm/console.php tools migration "/tmp/backup.sql"
```

Перед продолжением убедитесь, что восстановление завершилось **без ошибок**.

---

### 3. Запуск миграции

После успешного восстановления запустите миграцию одним из следующих способов.

#### 🧩 Вариант 1 — Командная строка (рекомендуется)

```bash
/home/xc_vm/bin/php/bin/php /home/xc_vm/console.php migrate
```

#### 🌐 Вариант 2 — Веб-установщик

* Вернитесь в **веб-установщик** (ссылка отображается при настройке панели)
* Выберите пункт **Migration**
* Следуйте инструкциям на экране

Вы будете видеть прогресс миграции в реальном времени.
После завершения система станет доступной.

## 🔑 Восстановление доступа после миграции

Если вход невозможен из-за отсутствующих учётных данных или кода доступа, используйте rescue-инструменты.

### Создание резервного кода доступа

```bash
php /home/xc_vm/console.php tools access
```

### Создание администратора

```bash
php /home/xc_vm/console.php tools user
```

> ⚠️ После восстановления доступа **немедленно смените** код доступа и учётные данные администратора.

## 🖥️ Подготовка балансировщиков нагрузки

Балансировщики нагрузки **не переносятся**.

* При необходимости переустановите операционную систему
* Перенастройте сеть и маршрутизацию
* Подключите их заново к основному серверу

## 🧩 Действия после миграции (обязательно)

После миграции система **не готова к продакшену**, пока не выполнены следующие шаги.

Пропуск этих действий приведёт к **ожидаемому, но неработающему поведению**.

---

### 1. Инициализация состояния выполнения

* Запустите все стримы вручную
* Убедитесь, что стримы доступны и работают стабильно

> Состояние выполнения стримов **никогда не сохраняется** при миграции.

---

### 2. Повторная настройка системы

Проверьте и восстановите все настройки, зависящие от окружения:

* Пути к файлам
* Лимиты и квоты
* Сетевые настройки и reverse proxy
* Параметры производительности

> **Не предполагается**, что значения по умолчанию совпадают с вашей прежней конфигурацией.
> Значения по умолчанию применяются намеренно.

---

### 3. Восстановление API-ключей и провайдеров

#### API-ключи никогда не переносятся

Следующее **необходимо настроить вручную**:

* **TMDb API key**

Это **ожидаемое поведение**.

> Если после миграции не загружаются метаданные,
> проверьте, что API-ключ добавлен заново и провайдер включён.
> Это **не является ошибкой миграции**.

---

## ❗ Распространённые проблемы после миграции

### Метаданные не загружаются (TMDb)

**Причина:**
API-ключ TMDb и конфигурация провайдера не были восстановлены.

**Решение:**
Повторно добавьте API-ключ TMDb и включите провайдер в настройках основного сервера.

---

## ✅ Итог

* Миграция переносит **только основные данные приложения**
* Конфигурация **исключена намеренно**
* API-ключи и настройки окружения **должны быть восстановлены вручную**
* Отсутствие функциональности после миграции **ожидаемо до завершения повторной настройки**
